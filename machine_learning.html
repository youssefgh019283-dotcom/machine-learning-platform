Supervised Learning

The algorithm learns a mapping Y = f(X) from labeled training data — where both inputs and correct outputs are provided.

🎓
What is Supervised Learning?
Given a dataset of (input, label) pairs, the algorithm adjusts its internal parameters to minimize prediction error. Once trained, it can predict labels for new, unseen inputs.
1
Collect labeled data — pairs of (input X, correct output Y)
2
Train the model — minimize a loss function via OLS or gradient descent
3
Validate — evaluate generalization on a held-out test set
4
Predict — apply to new data
Linear RegressionLogistic RegressionKNNDecision TreeNaive BayesSVMNeural Networks

Two main tasks

  • Regression — output is a continuous number (e.g., house price, temperature)
  • Classification — output is a discrete class label (e.g., spam/ham, cat/dog)

Key metrics

  • Regression: MSE, RMSE, MAE, R²
  • Classification: Accuracy, Precision, Recall, F1, AUC-ROC

📈 Simple Linear Regression

Find the best-fit line through data points to predict a continuous output from a single input feature.

Core Idea

Assume a linear relationship Y ≈ β₀ + β₁X. Find coefficients that minimize total squared error (residuals).

ŷ = β₀ + β₁·X
ŷ = prediction  |  β₀ = intercept  |  β₁ = slope
Loss = (1/n)·Σ(yᵢ − ŷᵢ)²
Mean Squared Error — what we minimize

OLS Closed-form Solution

  • β₁ = Σ(xᵢ−x̄)(yᵢ−ȳ) / Σ(xᵢ−x̄)²
  • β₀ = ȳ − β₁·x̄

R² — Coefficient of Determination

R² = 1 − SS_res/SS_tot. Ranges 0→1. Higher = better fit. 1 = perfect, 0 = no better than mean.

📌 Assumptions

Linearity, independence, homoscedasticity (constant variance), normality of residuals.

📌 Gradient Descent Alt.

Iteratively update β₁ -= α·∂L/∂β₁ until convergence. Works when OLS is too expensive.

📌 When to use

Baseline prediction, continuous target, when interpretability matters more than raw performance.

🔢 Multiple Linear Regression

Extend to multiple input features — the line becomes a hyperplane in n-dimensional feature space.

ŷ = β₀ + β₁X₁ + β₂X₂ + ··· + βₙXₙ
Each feature Xᵢ has its own coefficient βᵢ representing its independent contribution

Matrix Form

ŷ = X·β

OLS closed-form solution:

β = (XᵀX)⁻¹ Xᵀy

Key Considerations

  • Multicollinearity — correlated features inflate coefficient variance (check VIF)
  • Feature scaling — standardize for comparable coefficients
  • Adjusted R² — penalizes adding useless features
  • Ridge / Lasso — regularization to prevent overfitting

🔀 Logistic Regression

Despite the name, this is a classification algorithm. It uses the sigmoid function to output probabilities of belonging to a class.

Why Not Linear for Classification?

A linear model can output values outside [0,1]. We need probabilities — so we squash the linear output through the sigmoid function.

σ(z) = 1 / (1 + e⁻ᶻ)
Sigmoid squashes any real number to (0, 1)
z = β₀ + β₁X₁ + β₂X₂
Linear combination fed into sigmoid
Loss = −Σ[yᵢlog(ŷᵢ) + (1−yᵢ)log(1−ŷᵢ)]
Binary Cross-Entropy Loss — minimized via Gradient Descent

Decision Rule

  • If P(y=1|X) ≥ 0.5 → Class 1
  • If P(y=1|X) < 0.5 → Class 0
  • Threshold can be tuned for precision/recall tradeoff

🎯 K-Nearest Neighbors (KNN)

The simplest ML idea: a new point gets the class of its K closest neighbors. No training phase — it memorizes everything.

The Algorithm

1
Store all training data — no model, just memory
2
Compute distances — to every stored point
3
Find K nearest — sort by distance, take K closest
4
Majority vote — assign the most common class
d(p,q) = √Σ(pᵢ − qᵢ)²
Euclidean distance (Minkowski with p=2)

Choosing K

  • Small K → overfitting, jagged boundaries
  • Large K → underfitting, smoother boundaries
  • Rule of thumb: K = √n, use odd K for binary
  • Always scale features before using KNN!

💡 Weighted KNN

Closer neighbors vote more: weight = 1/d². Improves accuracy near decision boundaries.

🌳 Decision Tree

A tree of yes/no questions about features. Each split maximizes information gain until reaching pure leaf nodes.

Gini = 1 − Σ pᵢ²
0 = perfectly pure node, 0.5 = maximum disorder (binary)
Entropy = −Σ pᵢ·log₂(pᵢ)
Information theoretic impurity measure
IG = H(parent) − Σ wₖ·H(childₖ)
Information Gain — what we maximize at each split

Stopping Criteria

  • max_depth reached
  • Node has fewer than min_samples
  • No split improves impurity
  • All samples belong to same class

📊 Naive Bayes Classifier

A probabilistic classifier built on Bayes' theorem with a "naive" assumption that all features are conditionally independent given the class.

Bayes' Theorem

Update our prior belief about a class after observing the data features:

P(C|X) = P(X|C)·P(C) / P(X)
Posterior ∝ Likelihood × Prior

Each term explained

  • P(C|X) — posterior: P(class | features) — what we want
  • P(X|C) — likelihood: P(features | class) — from training data
  • P(C) — prior: how common is class C in training data?
  • P(X) — evidence: normalizing constant, same for all classes

The "Naive" Independence Assumption

P(X|C) = P(x₁|C)·P(x₂|C)·…·P(xₙ|C)

Almost never truly true, but works surprisingly well — especially for text!

Variants

  • Gaussian NB — continuous features, assumes Gaussian distribution
  • Multinomial NB — text/count features
  • Bernoulli NB — binary features
🍄 Spam Detection — Worked Example

Training: 100 emails — 60 ham, 40 spam

"Get FREE money now!"
P(Spam) = 40/100 = 0.40
P("FREE"|Spam) = 35/40 = 0.875
P("money"|Spam) = 30/40 = 0.75
Scorespam ∝ 0.40 × 0.875 × 0.75 = 0.263
P(Ham) = 60/100 = 0.60
P("FREE"|Ham) = 4/60 = 0.067
P("money"|Ham) = 2/60 = 0.033
Scoreham ∝ 0.60 × 0.067 × 0.033 = 0.0013
0.263 ≫ 0.0013 → 🚫 SPAM DETECTED

✅ Pros

Fast, small data friendly, great for text, easy to update with new data.

❌ Cons

Independence assumption broken in reality. Zero-frequency problem (fix with Laplace smoothing +1).

Unsupervised Learning

Find hidden patterns in unlabeled data. No correct answers are given — the algorithm must discover structure on its own.

🔍
What is Unsupervised Learning?
Given only inputs X (no labels Y), discover hidden patterns — clusters, low-dimensional structure, density regions, or association rules.
1
Raw unlabeled data — just inputs, no targets
2
Discover structure — algorithm finds natural groupings or components
3
Human interprets — domain expert makes sense of discovered patterns
Categories of Unsupervised Learning

Clustering

Group similar data points. K-Means, DBSCAN, Hierarchical. Used in customer segmentation, anomaly detection.

Dimensionality Reduction

Compress data while preserving structure. PCA, t-SNE, Autoencoders. Used for visualization, denoising.

Density Estimation

Model the underlying data distribution. GMM, KDE. Used for anomaly detection, generative models.

🔵 K-Means Clustering

Partition n observations into K clusters where each observation belongs to the cluster with the nearest centroid.

Lloyd's Algorithm

1
Initialize K centroids — random selection or K-Means++
2
Assign — each point → nearest centroid
3
Update — move centroid to mean of its cluster
4
Repeat 2–3 until convergence (no assignment changes)
J = Σₖ Σ_{x∈Cₖ} ‖x − μₖ‖²
Minimize WCSS — Within-Cluster Sum of Squares

Elbow Method for Choosing K

Plot WCSS vs K. Pick K at the "elbow" where improvement slows down.

Limitations

  • Assumes spherical clusters of similar size
  • Sensitive to initialization (use K-Means++)
  • K must be specified in advance
  • Sensitive to outliers

🧬 Principal Component Analysis (PCA)

Find the orthogonal axes of maximum variance in high-dimensional data. Project onto fewer dimensions while preserving as much structure as possible.

Intuition

If you have 100 correlated features, most information lives in a few directions. PCA finds those directions (eigenvectors of the covariance matrix) and ranks them by how much variance they capture (eigenvalues).

Step-by-step

1
Center the data — subtract mean from each feature
2
Covariance matrix — C = XᵀX / (n−1)
3
Eigendecomposition — C·v = λ·v
4
Sort by eigenvalue — PC1 has largest λ (most variance)
5
Project — X_reduced = X · [v₁, v₂, …, vₖ]

🌐 DBSCAN

Density-Based Spatial Clustering of Applications with Noise — finds arbitrarily shaped clusters and identifies outliers as noise.

Core Concepts

  • ε (epsilon) — neighborhood radius: how far to look for neighbors
  • minPts — minimum points in ε-neighborhood to form a dense region
  • Core Point — has ≥ minPts neighbors within ε
  • Border Point — within ε of a core point, but not itself a core
  • Noise Point — neither core nor border → outlier

Algorithm

1
For each unvisited point — find all ε-neighbors
2
If core point — start new cluster, expand recursively
3
Add reachable points — density-connected neighbors join cluster
4
Otherwise — mark as noise (can become border later)

vs K-Means

  • No need to specify K in advance
  • Handles arbitrary-shaped clusters
  • Robust to outliers (labels them as noise)
  • Struggles with varying density clusters

🌲 Hierarchical Clustering

Build a tree (dendrogram) of nested clusters by iteratively merging or splitting groups — no need to specify K in advance.

Agglomerative (Bottom-Up) — Most Common

1
Start — each point is its own cluster (n clusters)
2
Find closest pair — compute pairwise distance matrix
3
Merge — combine the two closest clusters into one
4
Repeat until 1 cluster remains (full dendrogram)

Linkage Criteria — How to measure cluster distance?

  • Single linkage — min dist between any two points (chaining effect)
  • Complete linkage — max dist (compact clusters)
  • Average linkage — mean dist (balanced)
  • Ward's method — minimize total WCSS increase (best for spherical)

✅ Pros

No K needed upfront. Dendrogram shows all possible clusterings. Good for exploration and visualizing hierarchy.

❌ Cons

O(n³) complexity — slow for large n. Cannot undo merges. Sensitive to noise and distance metric choice.

🆚 vs K-Means

Hierarchical is deterministic (no random init), gives a full tree, but K-Means scales to millions of points while hierarchical struggles above ~10k.
